Git & GitHub – Complete Learning Notes

These notes are designed for beginners who want to learn Git and GitHub for software development and team collaboration.

1. Introduction to Version Control

Version control is a system that helps track changes in files over time. It allows developers to manage code history and collaborate easily.

2. What is Git?

Git is a distributed version control system used to track changes in source code during software development.

Key Features of Git

  • Tracks code changes
  • Stores complete project history
  • Supports branching and merging
  • Works offline

3. Why Use Git?

  • Prevents code loss
  • Easy collaboration with teams
  • Rollback to previous versions
  • Industry standard tool

4. What is GitHub?

GitHub is a cloud-based platform that hosts Git repositories. It allows developers to store, manage, and share code online.

Features of GitHub

  • Online code hosting
  • Team collaboration
  • Issue tracking
  • Pull requests and code review

5. Git vs GitHub

Git GitHub
Version control system Code hosting platform
Runs locally Runs online
Manages code history Manages repositories

6. Basic Git Commands

  • git init – Initialize repository
  • git status – Check file status
  • git add . – Add files to staging
  • git commit -m "message" – Save changes
  • git push – Upload code to GitHub
  • git pull – Fetch latest updates
  • git clone – Copy repository

7. GitHub Workflow

  1. Create repository on GitHub
  2. Clone repository
  3. Make changes in code
  4. Commit changes
  5. Push code to GitHub

8. Advantages of Git & GitHub

  • Efficient code management
  • Easy collaboration
  • Project backup
  • Improves job opportunities

9. Conclusion

Git and GitHub are essential tools for every developer. Learning them helps in managing projects, collaborating with teams, and building a strong developer portfolio.
